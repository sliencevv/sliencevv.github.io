spring 简介

重要概念：控制反转与面向切面编程

  1. 依赖注入（DI）模式是控制反转的一个具体实现，控制反转（IOC）是一个通用的概念，它可以用很多不同的
    方式去表达，依赖注入是控制反转的一个具体例子。
  2. Spring 框架的一个关键组件是 面向方面的程序设计（AOP）框架。一个程序中跨越多个点的功能被称为 横切关
    注点，这些横切关注点在概念上独立于应用程序的业务逻辑。有各种各样常见的很好的关于方面的例子，比如日
    志记录、声明性事务、安全性，和缓存等等。
    在 OOP 中模块化的关键单元是类，而在 AOP 中模块化的关键单元是方面。AOP 帮助你将横切关注点从它们所
    影响的对象中分离出来，然而依赖注入帮助你将你的应用程序对象从彼此中分离出来。

体系结构

spring有可能成为所有企业应用程序的一站式服务点。spring是模块化的，允许你选择需要的使用。
spring结构图

(core containere)核心容器，有core(核心),bean,context(上下文),Sqel(表达式)。细节如下：

  1. 核心模块提供了框架的基本组成部分，包括IoC和依赖注入功能。
  2. bean模块提供BeanFactory,它是一个工厂模式的复杂实现。
  3. context 模块建立在core和bean模块提供的基础上，它是访问定义和配置的任何对象的媒介。
  4. 表达式语言模块在运行时提供了查询和操作一个对象图的强大的表达式语言。

(data access)数据访问/集成层，包括JDBC，ORM，OXM，JMS和事务处理模块。

  1. 因为目前使用的东西有限，这里只标记JDBC模块以及ORM模块。

web

  1. 目前使用的有spring web MVC。

其他

  1. AOP 模块提供了面向方面的编程实现，允许你定义方法拦截器和切入点对代码进行干净地解耦，它实现了应
    该分离的功能。
  2. Aspects 模块提供了与 J AspectJ 的集成，这是一个功能强大且成熟的面向切面编程（AOP）框架。
  3. 测试模块支持对具有 JUnit 或 TestNG 框架的 Spring 组件的测试。

分支管理

分支就像一个平行空间，你可以在这个平行空间里面进行随意切换，最后，你还可以将这些平行空间合并。
git会把每次的提交串成一个时间线，这条时间线就是一个分支。
在git中，master是主分支，head严格来说并不是指向提交，而是指向master,master才是指向提交的，所以head指向主分支。

  1. 创建分支dev,然后切换到dev分支。
    git branch dev,
    git checkout dev
  2. 查看当前分支
    git branch
  3. 合并分支
    git merge dev
  4. 删除分支
    git branch -d dev
